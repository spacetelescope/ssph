SSPH DEMO

Auth: $auth

You are: $eppn
IDP: $idp

  • The Login button is a GET to SSPH. SSPH runs as a CGI in an Apache server that is protected by SSO.
  • execution of the SSPH CGI means the authentication was successful
  • SSPH inserts a record of the authentication into the database
  • The SP checks for the authentication one of these two ways:
    • via HTTP (with mutual shared secret authentication)
    • by looking into its own database (if SSPH is configured to do put answers there)

    The SSPH prototype on etcbrady.stsci.edu is using a sqlite3 database, but it has also been used with Microsoft SQL server and MySQL.

    This demo is using a MySQL database running on banana.stsci.edu. In Database Confirmation mode, SSPH is inserting the authentication information directly into the applications database. In CGI Confirmation mode, the client contacts a CGI running on the SSPH server to confirm authentications.

    Protected Content is only visible when you log in.

    Source Code for the demo is only visible when you log in.

    ssph admin is a link to the SSPH admin page. The list of administrators is in the source code.
